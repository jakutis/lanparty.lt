# API package

An HTTP server written in Go language.

Package-wide convention: all logic and all error conditions are extensively logged.

## Running

Using `make`:
```bash
PORT=8080 OPENROUTER_API_KEY=... OPENROUTER_MODEL=... make run
```

Alternatively, using `go run`:
```bash
PORT=8080 OPENROUTER_API_KEY=... OPENROUTER_MODEL=... go run ./src
```

```bash
curl -XPOST localhost:8080/v1/representation -d '{"type":"html","spec":"a hello world web page"}'
```

`PORT` is optional and defaults to `8080`; it selects the TCP port the server
listens on. The `OPENROUTER_*` variables are documented in the
[Generator configuration](./main/generator.md#configuration).

## Endpoints

All endpoints are prefixed with /v1.

### POST /representation

Request entity template:
```json
{
  "type": "html",
  "spec": "Some prompt"
}
```

Both fields are required. Leading and trailing whitespace is trimmed from both
fields before validation and use.

On success the response status is `200 OK` and the response entity is a file generated by an LLM via OpenRouter from the given spec and type.

The response sets two headers derived from the requested `type`:

- `Content-Type`, the MIME type for the requested `type` (e.g. `html` ‚Üí `text/html; charset=utf-8`).
- `Content-Disposition: attachment; filename="representation.<ext>"`, where `<ext>` is the file extension for the requested `type` (e.g. `html` ‚Üí `representation.html`).

Only two `type` values are supported, matched case-insensitively:

| `type`     | Content-Type                   | Extension |
| ---------- | ------------------------------ | --------- |
| `html`     | `text/html; charset=utf-8`     | `.html`   |
| `markdown` | `text/markdown; charset=utf-8` | `.md`     |

Any other `type` is rejected with `422 Unprocessable Entity` before the
Generator is invoked.

#### Errors

Error responses carry a JSON entity of the shape `{"error":"<message>"}` with
`Content-Type: application/json; charset=utf-8`:

- `400 Bad Request` ‚Äî the request body is not a single valid JSON object
  matching the template: malformed JSON, unknown fields, trailing content
  after the object, or a body larger than 1 MiB.
- `422 Unprocessable Entity` ‚Äî `type` or `spec` is missing or empty
  (after whitespace trimming), or `type` is not one of the supported values.
- `500 Internal Server Error` ‚Äî the Generator failed to produce content.

Requests with a method other than `POST` receive `405 Method Not Allowed`.

## Generator

The representation endpoint delegates file generation to a [Generator](./main/generator.md): an LLM-backed component that produces raw file content for a given type and spec from the OpenRouter API.
